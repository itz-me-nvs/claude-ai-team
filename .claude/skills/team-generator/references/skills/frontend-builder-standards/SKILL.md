# Frontend Builder Standards

Stack: **Next.js 15 + React + TypeScript + Tailwind CSS v4 + shadcn/ui**

## Step 1 — Project Setup

Check if shadcn already initialised:
```bash
ls components/ui 2>/dev/null && echo "already set up — skip to Step 2" || echo "needs init"
```

**New Next.js project:**
```bash
npx shadcn@latest init -t next -y
```

**Existing Next.js project:**
```bash
npx shadcn@latest init -y
```

**Vite/React project:**
```bash
npx shadcn@latest init -t vite -y
```

`-t` flag wires Tailwind config for the correct framework. `-y` skips prompts (CSS variables on, radix base).

### Tailwind v4 CSS — correct globals.css structure

After init, verify `globals.css` / `index.css` looks like this:

```css
/* ✅ Correct */
@import "tailwindcss";       /* v4 engine */
@import "tw-animate-css";    /* animation utilities */

@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --color-primary: var(--primary);
  /* ...rest generated by shadcn init... */
}

:root {
  --background: oklch(...);
  --foreground: oklch(...);
  --primary: oklch(...);
  /* ...shadcn CSS variables... */
}
```

**❌ Never add — will break the build:**
```css
@import "shadcn/dist/tailwind.css";  /* shadcn ships NO CSS — this path doesn't exist */
@tailwind base;                       /* Tailwind v3 syntax — wrong for v4 */
@tailwind components;
@tailwind utilities;
```

shadcn is a CLI tool only. It generates component files and injects `:root {}` variables into your CSS file. There is nothing to import from shadcn.

## Step 2 — shadcn MCP Lookup (before every component)

```
mcp__shadcn__list_items_in_registries
mcp__shadcn__get_item_examples_from_registries  { "item": "<component-name>" }
```

Install:
```bash
npx shadcn@latest add <component> --yes --overwrite
```

**Component map:**
| Need | shadcn component |
|------|-----------------|
| Form fields | `form`, `input`, `select`, `checkbox`, `radio-group`, `switch`, `textarea` |
| Data display | `table`, `card`, `badge`, `avatar`, `separator` |
| Navigation | `sidebar`, `breadcrumb`, `tabs`, `navigation-menu`, `pagination` |
| Feedback | `toast`, `alert`, `dialog`, `alert-dialog`, `skeleton` |
| Layout | `sheet`, `drawer`, `collapsible`, `scroll-area`, `resizable` |
| Charts | `chart` (wraps Recharts) |

## Step 3 — Hard Rules (no exceptions)

### Components
- All UI primitives from shadcn — `<Button>`, `<Input>`, `<Select>`, `<Dialog>`, `<Toast>`, etc.
- Never reimplement what shadcn provides
- Extend with Tailwind classes — never override shadcn styles with custom CSS

### Loading states
- `<Skeleton>` from shadcn for ALL loading states
- Never use custom shimmer, `animate-pulse` divs, or custom `<Spinner>` components
- Inline micro-actions only: `<Loader2 className="animate-spin" />` (lucide-react)

```tsx
// ✅
import { Skeleton } from '@/components/ui/skeleton'
<Skeleton className="h-6 w-48" />

// ❌
<div className="animate-pulse bg-muted rounded h-6 w-48" />
```

### Forms
- Always `react-hook-form` + `zodResolver` — never manual `useState` validation
- Always shadcn `<Form>`, `<FormField>`, `<FormItem>`, `<FormLabel>`, `<FormControl>`, `<FormMessage>`

```tsx
const form = useForm<z.infer<typeof schema>>({
  resolver: zodResolver(schema),
  defaultValues: { ... },
})
```

### Theme
- Colors via shadcn CSS variables only: `--background`, `--foreground`, `--primary`, `--muted`, etc.
- Never hardcode hex or rgb values in components
- Dark mode via `.dark` class on `<html>` — shadcn handles this automatically
